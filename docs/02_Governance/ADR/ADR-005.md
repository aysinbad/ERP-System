## ADR-005 — أوامر التشغيل بوضعين (BOM vs Yield)
**السياق:** بعض أوامر التشغيل تصنيع (مكوّنات → منتج تام)، وبعضها فرزة خام (مدخل خام → مخرج صالح بلا منتج تام).
**القرار:** حقل `wo.mode`:
- `bom`: الهدر = الاستهلاك الفعلي − المعياري (تصنيع).
- `yield`: الهدر = المستلم − المنتَج الصالح (فرزة خام).
**البديل المرفوض:** صيغة واحدة `max(0, actual − standard)` للكل.
**السبب:** الصيغة الواحدة أعطت هدر = 0 خطأً في حالة الفرزة (14625 معياري مقابل 10000 فعلي). الفرزة تحتاج منطقاً معكوساً.
**الأثر:** `workOrderLines()` تحسب حسب الـ mode. `woProduces()` ترجع false في وضع yield (لا منتج تام) فالقيد يمشي في مسار "هدر فقط".
